## Getting Started

### Prerequisites

Before you begin, make sure you have:

- **Node.js** (v16 or higher)
- **pnpm** (v8.2 or higher)
- A GitHub account

### Development Setup

You can develop locally or use GitHub Codespaces to get started quickly.

#### Option 1: Develop Locally

1. **Fork the repository** - Click the Fork button on the [Starlight repository](https://github.com/withastro/starlight)

2. **Clone your fork** - Replace `YOUR-USERNAME` with your GitHub username:

   ```bash
   git clone https://github.com/YOUR-USERNAME/starlight.git
   cd starlight
   ```

3. **Install dependencies**:

   ```bash
   pnpm i
   ```

4. **Generate TypeScript types**:

   ```bash
   cd docs
   pnpm astro sync
   ```

5. **Start the dev server**:

   ```bash
   pnpm dev
   ```

   Visit `http://localhost:4321` to see your changes.

#### Option 2: Use GitHub Codespaces

1. Create a new codespace at [codespaces.new/withastro/starlight](https://codespaces.new/withastro/starlight)
2. Generate types: `pnpm astro sync`
3. Start the dev server: `cd docs && pnpm dev --host`

## Repository Structure

Starlight is a monorepo containing several projects:

- **`docs/`** - The Starlight documentation site
- **`packages/starlight/`** - The main Starlight integration package
- **`packages/`** - Additional Starlight packages
- **`examples/`** - Example projects using Starlight

## Making a Pull Request

### Adding a Changeset

When making changes that impact users (changes to `packages/*`), add a changeset:

```bash
pnpm exec changeset
```

Documentation-only changes (`docs/*`) and examples (`examples/*`) do not need changesets.

### Pull Request Guidelines

- Write clear, descriptive commit messages
- Include tests for new features
- Update documentation as needed
- Follow the existing code style
- Keep PRs focused on a single issue or feature

## Testing

### Visual Changes

Test your changes by running the docs dev server:

```bash
cd docs
pnpm dev
```

**Note:** Changes to the Starlight integration require restarting the dev server.

### Link Checking

Check for broken links in the documentation:

```bash
cd docs
pnpm linkcheck
```

### Unit Tests

Run unit tests with Vitest:

```bash
cd packages/starlight
pnpm test
```

For test coverage:

```bash
pnpm test:coverage
```

### End-to-End Tests

Run E2E tests with Playwright:

```bash
cd packages/starlight
pnpm test:e2e
```

## Translation Contributions

Translations help make Starlight accessible to more people. You can contribute by:

### Translating Starlight's UI

Add or update translation files in `packages/starlight/translations/`. Each language's JSON file follows the [translation structure in the docs](https://starlight.astro.build/guides/i18n/#translate-starlights-ui).

Watch the [video introduction to Starlight's translation files](https://scrimba.com/scrim/cpb44bt3) for a visual walkthrough.

### Translating Documentation

Help translate the Starlight documentation:

- Review [open translation PRs](https://github.com/withastro/starlight/pulls)
- Update out-of-date translated pages
- Add untranslated pages

Track translation progress at [i18n.starlight.astro.build](https://i18n.starlight.astro.build).

## Contributing to the Showcase

### Add Your Site

Built a site with Starlight? Add it to the [showcase](https://starlight.astro.build/resources/showcase/):

1. Add a screenshot (800×450px PNG) to `docs/src/assets/showcase/`
2. Name it after your domain: `example.com.png`
3. Add your site entry to `docs/src/components/showcase-sites.astro`
4. Open a pull request

### Add Your Theme

Created a theme for Starlight? Share it on the [themes page](https://starlight.astro.build/resources/themes/):

1. Take screenshots of light and dark modes (1280×720px)
2. Add screenshots to `docs/src/assets/themes/` as `theme-name-light.png` and `theme-name-dark.png`
3. Add your theme entry to `docs/src/content/docs/resources/themes.mdx`
4. Open a pull request

## Understanding Starlight

### Architecture

- **Astro Integration** - Starlight is built as an Astro integration (see [`packages/starlight/index.ts`](https://github.com/withastro/starlight/blob/main/packages/starlight/index.ts))
- **Single Route Template** - Most pages use [`packages/starlight/index.astro`](https://github.com/withastro/starlight/blob/main/packages/starlight/index.astro)
- **Content Collections** - Content comes from the `docs` [content collection](https://docs.astro.build/en/guides/content-collections/)
- **Framework-Agnostic** - Components use custom elements instead of framework-specific code

### Browser Support

Starlight supports [modern browsers](https://browsersl.ist/#q=%3E+0.5%25%2C+not+dead%2C+Chrome+%3E%3D+105%2C+Edge+%3E%3D+105%2C+Firefox+%3E%3D+121%2C+Safari+%3E%3D+15.4%2C+iOS+%3E%3D+15.4%2C+not+op_mini+all). Use [Can I use](https://caniuse.com) to check feature compatibility.
