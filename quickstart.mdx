This guide will walk you through creating a new Starlight documentation site from scratch. You'll go from zero to a fully functional documentation site in just a few minutes.

<Note>
  Already have an Astro project? Check out the [Manual Setup](/manual-setup) guide to add Starlight to your existing project.
</Note>

## Prerequisites

Before you begin, make sure you have:

- **Node.js** version 18.17.1 or higher
- A terminal/command line interface
- A text editor (we recommend VS Code with the Astro extension)

## Create a New Project

<Steps>
  <Step title="Run the create command">
    Use your preferred package manager to create a new Starlight project. This command will launch an interactive wizard to help you set up your project.
    
    <CodeGroup>
      ```bash npm
      npm create astro@latest -- --template starlight
      ```
      
      ```bash pnpm
      pnpm create astro --template starlight
      ```
      
      ```bash yarn
      yarn create astro --template starlight
      ```
    </CodeGroup>
  </Step>
  
  <Step title="Answer the setup questions">
    The wizard will ask you a few questions:
    
    - **Where should we create your new project?** — Enter a directory name (e.g., `my-docs`)
    - **Install dependencies?** — Yes (recommended)
    - **Initialize a git repository?** — Yes (recommended)
    
    Once complete, you'll see a success message with next steps.
  </Step>
  
  <Step title="Navigate to your project">
    Change into your new project directory:
    
    ```bash
    cd my-docs
    ```
  </Step>
</Steps>

## Start the Development Server

Now let's start the development server to see your site in action:

<CodeGroup>
  ```bash npm
  npm run dev
  ```
  
  ```bash pnpm
  pnpm dev
  ```
  
  ```bash yarn
  yarn dev
  ```
</CodeGroup>

You should see output similar to this:

```bash
🚀 astro v5.6.1 started in 45ms

┃ Local    http://localhost:4321/
┃ Network  use --host to expose
```

Open **http://localhost:4321** in your browser. You'll see your new Starlight documentation site with:

- A homepage with a hero section
- Sample documentation pages
- Automatic sidebar navigation
- Built-in search functionality
- Dark mode toggle

<Note>
  The development server watches your files and automatically refreshes the browser when you make changes. Keep it running while you work!
</Note>

## Explore the Project Structure

Your new Starlight project has the following structure:

```
my-docs/
├── src/
│   ├── content/
│   │   ├── docs/           # Your documentation pages
│   │   │   └── index.mdx   # Homepage
│   │   └── config.ts       # Content collections config
│   └── content.config.ts   # Content configuration
├── public/                 # Static assets
├── astro.config.mjs        # Astro & Starlight config
├── package.json
└── tsconfig.json
```

### Key Files and Directories

- **`src/content/docs/`** — All your documentation pages go here. Each Markdown or MDX file becomes a page.
- **`astro.config.mjs`** — Main configuration file where Starlight is configured.
- **`src/content/config.ts`** — Content collections schema configuration.
- **`public/`** — Static files like images, fonts, or downloads.

## Add Your First Page

<Steps>
  <Step title="Create a new Markdown file">
    Create a new file at `src/content/docs/getting-started.md`:
    
    ```md
    ---
    title: Getting Started
    description: Learn the basics of using our product.
    ---
    
    Welcome to the getting started guide!
    
    ## Installation
    
    Install our package using npm:
    
    ```bash
    npm install your-package
    ```
    
    ## Basic Usage
    
    Here's a simple example:
    
    ```js
    import { feature } from 'your-package';
    
    feature.doSomething();
    ```
    ```
  </Step>
  
  <Step title="View your new page">
    Save the file and navigate to **http://localhost:4321/getting-started** in your browser. Your new page appears automatically in the sidebar!
  </Step>
</Steps>

## Customize Your Site

Let's personalize your documentation site by editing the configuration:

<Steps>
  <Step title="Open the config file">
    Open `astro.config.mjs` in your editor. You'll see the Starlight configuration:
    
    ```js
    // astro.config.mjs
    import { defineConfig } from 'astro/config';
    import starlight from '@astrojs/starlight';
    
    export default defineConfig({
      integrations: [
        starlight({
          title: 'My Docs',
          social: {
            github: 'https://github.com/withastro/starlight',
          },
          sidebar: [
            {
              label: 'Guides',
              items: [
                { label: 'Example Guide', slug: 'guides/example' },
              ],
            },
          ],
        }),
      ],
    });
    ```
  </Step>
  
  <Step title="Update the title and social links">
    Modify the title and add your own social media links:
    
    ```js
    starlight({
      title: 'My Product Docs',
      social: {
        github: 'https://github.com/yourusername/yourproject',
        twitter: 'https://twitter.com/yourhandle',
      },
      // ... rest of config
    })
    ```
  </Step>
  
  <Step title="Save and see the changes">
    Save the file and your browser will automatically refresh with the updated title and social links in the header.
  </Step>
</Steps>

## Build for Production

When you're ready to deploy your documentation site:

<CodeGroup>
  ```bash npm
  npm run build
  ```
  
  ```bash pnpm
  pnpm build
  ```
  
  ```bash yarn
  yarn build
  ```
</CodeGroup>

This creates an optimized production build in the `dist/` directory. You can preview the production build locally:

<CodeGroup>
  ```bash npm
  npm run preview
  ```
  
  ```bash pnpm
  pnpm preview
  ```
  
  ```bash yarn
  yarn preview
  ```
</CodeGroup>

## Deploy Your Site

Starlight sites can be deployed to any static hosting platform:

<CardGroup cols={2}>
  <Card title="Netlify" icon="cloud">
    Connect your Git repository and Netlify will automatically build and deploy your site.
  </Card>
  
  <Card title="Vercel" icon="cloud">
    Import your project and Vercel handles the rest with zero configuration.
  </Card>
  
  <Card title="GitHub Pages" icon="github">
    Deploy directly from your GitHub repository using GitHub Actions.
  </Card>
  
  <Card title="Cloudflare Pages" icon="cloud">
    Deploy with automatic builds triggered by Git commits.
  </Card>
</CardGroup>

<Warning>
  Remember to set your site's `base` in `astro.config.mjs` if deploying to a subdirectory (e.g., GitHub Pages with a project repository).
</Warning>
