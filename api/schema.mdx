Starlight provides schema utilities to define and extend content collection schemas for your documentation pages and internationalization files.

## `docsSchema()`

Defines the content collection schema for Starlight's `docs` collection.

### Signature

```ts
function docsSchema<T extends BaseSchema = never>(
  options?: DocsSchemaOpts<T>
): (context: SchemaContext) => ExtendedSchema<T>
```

### Parameters

#### `options`

**Type:** `DocsSchemaOpts<T>` (optional)

An optional configuration object with the following property:

##### `extend`

**Type:** `BaseSchema | ((context: SchemaContext) => BaseSchema)` (optional)

Extends Starlight's default schema with additional fields. Can be either a Zod schema object or a function that receives the schema context and returns a Zod schema.

Supported Zod types:
- `z.AnyZodObject`
- `z.ZodUnion`
- `z.ZodDiscriminatedUnion`
- `z.ZodIntersection`
- `z.ZodEffects` wrapping any of the above

### Returns

A function that takes a `SchemaContext` and returns the extended schema combining Starlight's default frontmatter fields with your custom fields.

### Default Schema Fields

The default schema includes these frontmatter fields:

- **`title`** (string, required) - The title of the page
- **`description`** (string, optional) - A short description of the page content (150-160 characters recommended)
- **`editUrl`** (string URL | boolean, optional, default: `true`) - Custom edit URL or `false` to disable
- **`head`** (array, optional) - Custom `<head>` tags for this page
- **`tableOfContents`** (object | boolean, optional) - Override table of contents configuration
- **`template`** (`'doc'` | `'splash'`, default: `'doc'`) - Layout template for the page
- **`hero`** (object, optional) - Hero section configuration
- **`lastUpdated`** (Date | boolean, optional) - Override last update date or disable it
- **`prev`** (object, optional) - Previous page navigation link configuration
- **`next`** (object, optional) - Next page navigation link configuration
- **`sidebar`** (object, optional) - Sidebar configuration with:
  - `order` (number) - Sort order in navigation
  - `label` (string) - Custom label in navigation
  - `hidden` (boolean, default: `false`) - Hide from autogenerated sidebar groups
  - `badge` (string | object) - Badge to display on sidebar link
  - `attrs` (object) - HTML attributes for sidebar link
- **`banner`** (object, optional) - Announcement banner with:
  - `content` (string) - Banner content (supports HTML)
- **`pagefind`** (boolean, default: `true`) - Enable/disable Pagefind indexing
- **`draft`** (boolean, default: `false`) - Mark as draft (excluded from production builds)

### Examples

<Tabs>
<TabItem label="Basic Extension">

```ts title="src/content/config.ts"
import { defineCollection } from 'astro:content';
import { docsSchema } from '@astrojs/starlight/schema';
import { z } from 'astro/zod';

export const collections = {
  docs: defineCollection({
    schema: docsSchema({
      extend: z.object({
        // Add a category field
        category: z.enum(['tutorial', 'guide', 'reference']).optional(),
      }),
    }),
  }),
};
```

<TabItem label="Using Schema Context">

```ts title="src/content/config.ts"
import { defineCollection } from 'astro:content';
import { docsSchema } from '@astrojs/starlight/schema';
import { z } from 'astro/zod';

export const collections = {
  docs: defineCollection({
    schema: docsSchema({
      extend: ({ image }) => {
        return z.object({
          // Use Astro's image helper for cover images
          cover: image().optional(),
        });
      },
    }),
  }),
};
```

<TabItem label="Multiple Fields">

```ts title="src/content/config.ts"
import { defineCollection } from 'astro:content';
import { docsSchema } from '@astrojs/starlight/schema';
import { z } from 'astro/zod';

export const collections = {
  docs: defineCollection({
    schema: docsSchema({
      extend: z.object({
        // Add multiple custom fields
        category: z.enum(['tutorial', 'guide', 'reference']).optional(),
        author: z.string().optional(),
        tags: z.array(z.string()).default([]),
      }),
    }),
  }),
};
```

## `i18nSchema()`

Defines the content collection schema for Starlight's optional `i18n` collection used for custom UI translations.

### Signature

```ts
function i18nSchema<T extends z.AnyZodObject = z.SomeZodObject>(
  options?: i18nSchemaOpts<T>
): ExtendedSchema<T>
```

### Parameters

#### `options`

**Type:** `i18nSchemaOpts<T>` (optional)

An optional configuration object with the following property:

##### `extend`

**Type:** `z.AnyZodObject` (optional)

Extends Starlight's default i18n schema with additional translation keys. Must be a Zod object schema.

### Returns

A passthrough Zod schema that merges Starlight's built-in UI strings (including Pagefind and Expressive Code translations) with your custom translation keys.

### Default i18n Keys

The default schema includes translation keys for:

- **Skip link** - `skipLink.label`
- **Search UI** - `search.label`, `search.ctrlKey`, `search.cancelLabel`, `search.devWarning`
- **Theme selector** - `themeSelect.accessibleLabel`, `themeSelect.dark`, `themeSelect.light`, `themeSelect.auto`
- **Language selector** - `languageSelect.accessibleLabel`
- **Mobile menu** - `menuButton.accessibleLabel`
- **Sidebar navigation** - `sidebarNav.accessibleLabel`
- **Table of contents** - `tableOfContents.onThisPage`, `tableOfContents.overview`
- **Internationalization** - `i18n.untranslatedContent`
- **Page elements** - `page.editLink`, `page.lastUpdated`, `page.previousLink`, `page.nextLink`, `page.draft`
- **404 page** - `404.text`
- **Asides** - `aside.tip`, `aside.note`, `aside.caution`, `aside.danger`
- **File tree** - `fileTree.directory`
- **Footer** - `builtWithStarlight.label`
- **Headings** - `heading.anchorLabel`
- **Pagefind translations** - `pagefind.*` (see [Pagefind UI docs](https://pagefind.app/docs/ui/#translations))
- **Expressive Code translations** - `expressiveCode.copyButtonCopied`, `expressiveCode.copyButtonTooltip`, `expressiveCode.terminalWindowFallbackTitle`

All keys are optional by default.

### Example

```ts title="src/content/config.ts"
import { defineCollection } from 'astro:content';
import { i18nSchema } from '@astrojs/starlight/schema';
import { z } from 'astro/zod';

export const collections = {
  i18n: defineCollection({
    type: 'data',
    schema: i18nSchema({
      extend: z
        .object({
          // Add custom UI strings
          'customUi.heading': z.string(),
          'customUi.text': z.string(),
          'customUi.button': z.string(),
        })
        .partial(),
    }),
  }),
};
```

Then use your custom strings in translation files:

```json title="src/content/i18n/en.json"
{
  "customUi.heading": "Welcome",
  "customUi.text": "Get started with our documentation",
  "customUi.button": "Read more"
}
```
