---
title: Frontmatter Reference
description: Complete reference for all Starlight frontmatter fields.
---

Starlight pages support frontmatter fields to customize page behavior and metadata. This page documents all available frontmatter options.

## Required Fields

<ParamField path="title" type="string" required>
  The title of the current page. Used in the page header, browser tab, and metadata.
  
  ```yaml
  ---
  title: Getting Started
  ---
  ```
</ParamField>

## Metadata

<ParamField path="description" type="string">
  A short description of the current page's content. Optional, but recommended.
  
  A good description is 150–160 characters long and outlines the key content of the page in a clear and engaging way. Used for SEO and social media previews.
  
  ```yaml
  ---
  title: Getting Started
  description: Learn how to start building your documentation site with Starlight.
  ---
  ```
</ParamField>

<ParamField path="head" type="array">
  Set custom `<head>` tags just for this page. Useful for page-specific scripts, styles, or metadata.
  
  Each entry requires:
  - `tag`: HTML tag name (e.g., 'meta', 'link', 'script')
  - `attrs`: Object of HTML attributes
  
  ```yaml
  ---
  title: About
  head:
    - tag: meta
      attrs:
        property: og:image
        content: https://example.com/image.png
  ---
  ```
</ParamField>

## Page Layout

<ParamField path="template" type="'doc' | 'splash'" default="'doc'">
  Set the layout style for this page.
  
  - `doc`: The default layout with sidebar navigation
  - `splash`: A wider layout without any sidebars, ideal for landing pages
  
  ```yaml
  ---
  title: Welcome
  template: splash
  ---
  ```
</ParamField>

<ParamField path="hero" type="object">
  Display a hero section on this page. Typically used with the `splash` template.
  
  <ParamField path="hero.title" type="string">
    The large title text to show. If not provided, defaults to the top-level `title`. Can include HTML.
  </ParamField>
  
  <ParamField path="hero.tagline" type="string">
    A short bit of text about your project. Displayed in a smaller size below the title.
  </ParamField>
  
  <ParamField path="hero.image" type="object">
    The image to use in the hero.
    
    ### Single Image
    
    <ParamField path="hero.image.file" type="image">
      Relative path to an image file in your repo (e.g., '../../assets/hero.png').
    </ParamField>
    
    <ParamField path="hero.image.alt" type="string" default="''">
      Alt text for screenreaders and assistive technologies.
    </ParamField>
    
    ### Dark/Light Mode Images
    
    <ParamField path="hero.image.light" type="image">
      Relative path to an image file to use in light mode.
    </ParamField>
    
    <ParamField path="hero.image.dark" type="image">
      Relative path to an image file to use in dark mode.
    </ParamField>
    
    ### Raw HTML
    
    <ParamField path="hero.image.html" type="string">
      Raw HTML string instead of an image file. Useful for inline SVGs or more complex hero content.
    </ParamField>
  </ParamField>
  
  <ParamField path="hero.actions" type="array" default="[]">
    An array of call-to-action links displayed at the bottom of the hero.
    
    <ParamField path="hero.actions[].text" type="string" required>
      Text label displayed in the link.
    </ParamField>
    
    <ParamField path="hero.actions[].link" type="string" required>
      Value for the link's `href` attribute (e.g., '/page' or 'https://mysite.com').
    </ParamField>
    
    <ParamField path="hero.actions[].variant" type="'primary' | 'secondary' | 'minimal'" default="'primary'">
      Button style to use.
    </ParamField>
    
    <ParamField path="hero.actions[].icon" type="string | object">
      An optional icon to display alongside the link text. Can be an inline SVG or the name of a built-in icon.
    </ParamField>
    
    <ParamField path="hero.actions[].attrs" type="object">
      HTML attributes to add to the link.
    </ParamField>
  </ParamField>
  
  ```yaml
  ---
  title: Welcome to My Docs
  template: splash
  hero:
    title: Build Amazing Docs
    tagline: Fast, accessible, and beautiful documentation with Starlight
    image:
      file: ../../assets/hero.png
      alt: A vibrant illustration
    actions:
      - text: Get Started
        link: /quickstart
        variant: primary
      - text: View on GitHub
        link: https://github.com/withastro/starlight
        variant: secondary
  ---
  ```
</ParamField>

<ParamField path="banner" type="object">
  Display an announcement banner at the top of this page.
  
  <ParamField path="banner.content" type="string" required>
    The content of the banner. Supports HTML syntax.
  </ParamField>
  
  ```yaml
  ---
  title: Important Update
  banner:
    content: 'This documentation is for v2.0. <a href="/v1/">View v1 docs</a>'
  ---
  ```
</ParamField>

## Table of Contents

<ParamField path="tableOfContents" type="boolean | object">
  Override global table of contents configuration for this page. Set to `false` to hide the table of contents.
  
  <ParamField path="tableOfContents.minHeadingLevel" type="number" default="2">
    The level to start including headings at (1-6).
  </ParamField>
  
  <ParamField path="tableOfContents.maxHeadingLevel" type="number" default="3">
    The level to stop including headings at (1-6).
  </ParamField>
  
  ```yaml
  ---
  title: API Reference
  tableOfContents:
    minHeadingLevel: 2
    maxHeadingLevel: 4
  ---
  ```
</ParamField>

## Sidebar

<ParamField path="sidebar" type="object" default="{}">
  Configure how this page appears in the sidebar navigation.
  
  <ParamField path="sidebar.order" type="number">
    The order of this page in the navigation. Pages are sorted by this value in ascending order, then by slug. If not provided, pages are sorted alphabetically by slug.
  </ParamField>
  
  <ParamField path="sidebar.label" type="string">
    The label for this page in the navigation. Defaults to the page `title` if not set.
  </ParamField>
  
  <ParamField path="sidebar.hidden" type="boolean" default="false">
    Prevents this page from being included in autogenerated sidebar groups.
  </ParamField>
  
  <ParamField path="sidebar.badge" type="string | object">
    Adds a badge to the sidebar link. Can be a string or an object with `variant` and `text`.
    
    Variants include: 'note', 'tip', 'caution', 'danger', 'success', and 'default'.
  </ParamField>
  
  <ParamField path="sidebar.attrs" type="object">
    HTML attributes to add to the sidebar link.
  </ParamField>
  
  ```yaml
  ---
  title: Advanced Configuration
  sidebar:
    order: 3
    badge: New
  ---
  ```
</ParamField>

## Page Footer

<ParamField path="editUrl" type="string (URL) | boolean" default="true">
  Custom URL where a reader can edit this page. Overrides the `editLink.baseUrl` global config if set.
  
  Set to `false` to disable showing an edit link on this page.
  
  ```yaml
  ---
  title: Contributing Guide
  editUrl: https://github.com/myorg/myrepo/edit/main/docs/contributing.md
  ---
  ```
</ParamField>

<ParamField path="lastUpdated" type="Date | boolean">
  The last update date of the current page. Overrides the `lastUpdated` global config or the date generated from Git history.
  
  Set to `false` to hide the last updated date on this page.
  
  ```yaml
  ---
  title: Changelog
  lastUpdated: 2024-03-15
  ---
  ```
</ParamField>

<ParamField path="prev" type="boolean | string | object">
  The previous navigation link configuration. Overrides the `pagination` global config.
  
  - `false`: Hide the previous link
  - String: Use as the link URL
  - Object: Customize both URL and label
  
  <ParamField path="prev.link" type="string">
    The navigation link URL.
  </ParamField>
  
  <ParamField path="prev.label" type="string">
    The navigation link text.
  </ParamField>
  
  ```yaml
  ---
  title: Chapter 3
  prev:
    link: /chapter-2/
    label: Previous Chapter
  ---
  ```
</ParamField>

<ParamField path="next" type="boolean | string | object">
  The next navigation link configuration. Overrides the `pagination` global config.
  
  - `false`: Hide the next link
  - String: Use as the link URL
  - Object: Customize both URL and label
  
  <ParamField path="next.link" type="string">
    The navigation link URL.
  </ParamField>
  
  <ParamField path="next.label" type="string">
    The navigation link text.
  </ParamField>
  
  ```yaml
  ---
  title: Chapter 3
  next:
    link: /chapter-4/
    label: Next Chapter
  ---
  ```
</ParamField>

## Search and Indexing

<ParamField path="pagefind" type="boolean" default="true">
  Control Pagefind indexing for this page. Set to `false` to exclude this page from search results.
  
  ```yaml
  ---
  title: Internal Notes
  pagefind: false
  ---
  ```
</ParamField>

<ParamField path="draft" type="boolean" default="false">
  Indicates that this page is a draft and will not be included in production builds.
  
  **Note:** The page will still be available when running Astro in development mode.
  
  ```yaml
  ---
  title: Upcoming Features
  draft: true
  ---
  ```
</ParamField>
