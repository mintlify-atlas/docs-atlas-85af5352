Starlight provides content loaders for use with Astro's [Content Layer API](https://docs.astro.build/en/guides/content-collections/#defining-collections-with-loaders). These loaders automatically discover and load your documentation pages and i18n translation files.

## `docsLoader()`

Loads content files from the `src/content/docs/` directory. Automatically handles Markdown, MDX, and Markdoc files while ignoring files starting with `_`.

### Signature

```ts
function docsLoader(options?: {
  generateId?: GenerateIdFunction;
}): Loader
```

### Parameters

#### `options`

**Type:** `object` (optional)

Configuration options for the loader:

##### `generateId`

**Type:** `(entry: { filePath: string }) => string` (optional)

A function that generates a unique ID for each entry. By default, generates a slug from the entry's file path.

**Parameters:**
- `entry.filePath` (string) - The file path of the entry relative to the collection directory

**Returns:** A unique ID string for the entry.

### Returns

A loader object compatible with Astro's Content Layer API.

### Behavior

- **File patterns:** Matches files with extensions: `.markdown`, `.mdown`, `.mkdn`, `.mkd`, `.mdwn`, `.md`, `.mdx`
- **Markdoc support:** Automatically includes `.mdoc` files when `@astrojs/markdoc` integration is detected
- **Ignores files:** Starting with underscore (`_`) are excluded from loading
- **Base directory:** Loads from `src/content/docs/` by default

### Example

<Tabs>
<TabItem label="Basic Usage">

```ts title="src/content/config.ts"
import { defineCollection } from 'astro:content';
import { docsLoader } from '@astrojs/starlight/loaders';
import { docsSchema } from '@astrojs/starlight/schema';

export const collections = {
  docs: defineCollection({
    loader: docsLoader(),
    schema: docsSchema(),
  }),
};
```

</TabItem>
<TabItem label="Custom ID Generation">

```ts title="src/content/config.ts"
import { defineCollection } from 'astro:content';
import { docsLoader } from '@astrojs/starlight/loaders';
import { docsSchema } from '@astrojs/starlight/schema';

export const collections = {
  docs: defineCollection({
    loader: docsLoader({
      generateId: ({ filePath }) => {
        // Custom ID generation logic
        // For example, preserve directory structure in IDs
        return filePath.replace(/\.(md|mdx|mdoc)$/, '');
      },
    }),
    schema: docsSchema(),
  }),
};
```

## `i18nLoader()`

Loads internationalization data files from the `src/content/i18n/` directory. Supports JSON, YAML, and YML file formats while ignoring files starting with `_`.

### Signature

```ts
function i18nLoader(): Loader
```

### Parameters

This loader accepts no parameters.

### Returns

A loader object compatible with Astro's Content Layer API.

### Behavior

- **File patterns:** Matches files with extensions: `.json`, `.yml`, `.yaml`
- **Ignores files:** Starting with underscore (`_`) are excluded from loading
- **Base directory:** Loads from `src/content/i18n/` by default
- **Naming convention:** File names typically match locale codes (e.g., `en.json`, `fr.json`, `es.json`)

### Example

```ts title="src/content/config.ts"
import { defineCollection } from 'astro:content';
import { i18nLoader } from '@astrojs/starlight/loaders';
import { i18nSchema } from '@astrojs/starlight/schema';

export const collections = {
  i18n: defineCollection({
    type: 'data',
    loader: i18nLoader(),
    schema: i18nSchema(),
  }),
};
```

With this configuration, you can create translation files:

```json title="src/content/i18n/en.json"
{
  "page.editLink": "Edit this page",
  "page.lastUpdated": "Last updated:"
}
```

```json title="src/content/i18n/fr.json"
{
  "page.editLink": "Modifier cette page",
  "page.lastUpdated": "Dernière mise à jour :"
}
```

## Type Definitions

### `Loader`

The loader type from Astro's Content Layer API:

```ts
interface Loader {
  name: string;
  load: (context: LoaderContext) => Promise<void> | void;
}
```

### `GenerateIdFunction`

Function type for custom ID generation:

```ts
type GenerateIdFunction = (entry: { filePath: string }) => string;
```
